*version*
This field is automatically checked and refreshed, don't modify it manually.

*signsList*
All this file content is automatically generated, because it is necessary to save signs after server shutdown.
After creating signs in the game, you can also edit the file manually and use the command /tm reload signs.
Editing this file does not allow the creation of new signs. Only the 'lines' and the 'side' entries will be updated in the game. 
  *id*
   Each sign must have an unique id.
  *world*
   The name of the world where the sign is in.
  *position*
   If a sign exists at this position (x, y, z), its placeholders would be actualized.
  *material*
   This entry tells you the material of the sign, making it easier to recognize in the list.
  *face*
   The cardinal orientation of the sign, making it easier to recognize in the list.
  *side*
   The side of the sign where the text is placed (FRONT or BACK). Necessary to distinguish the two sides in two different entries of the list.
  *content*
    *line1*
     If this line contents the marker defined in the config.yml file (by default : [tm]), auto-refresh will be activated.
    *line2*
    *line3*
    *line4*
     In the following lines, any placeholder (formatted as : {tm_xxx}) will be replaced by a constantly updated value.
